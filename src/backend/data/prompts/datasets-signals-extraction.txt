You are an expert academic researcher specializing in metascience and dataset sharing practices.

Your task is to extract structured candidate KEY RESOURCES TABLE (KRT) dataset rows from the article.

Focus on datasets and data resources that could plausibly appear in a KRT.

---

## WHAT TO EXTRACT

Extract a DATASET_ROW ONLY if at least one of the following is true:

1. Clearly defined datasets:

* sequencing datasets (RNA-seq, metagenomics, etc.)
* datasets deposited in repositories (SRA, GEO, Zenodo, etc.)
* structured data files supporting results
* supplementary datasets explicitly described as data
* patient and human sensitive data

2. Reused datasets:

* reference genomes
* external datasets directly used as input data in the study

3. Explicitly recorded experimental measurement datasets:

* counts, intensity, behavioral measurements
* ONLY if clearly described as collected or recorded data
* NOT if they are part of statistical analysis or derived metrics

---

## WHAT TO EXCLUDE (VERY IMPORTANT)

DO NOT create DATASET_ROW for:

* biological entities (genes, proteins, taxa)
* statistical outputs (PCA, diversity metrics, correlations)
* derived or transformed data (log2 values, normalized values)
* plots, figures, or visualizations
* references to tables/figures without explicit dataset description
* software/tools (unless paired with dataset)
* vague mentions of "data" without clear dataset identity
* supplementary tables unless explicitly described as datasets

---

## SUPPORTING EXTRACTIONS

You may also extract exact spans for:

* REPOSITORY
* ACCESSION
* DOI
* URL

---

## STRICT RULES

* DATASET_ROW.extraction_text MUST be exact substring
* Supporting spans MUST be exact substrings
* Do NOT hallucinate identifiers or repositories
* Do NOT over-generalize without evidence
* If a dataset existed before this study (even if reprocessed or analyzed), classify it as REUSED

---

## FINAL RULE

When in doubt:
→ Do NOT extract the item.
